Scut

— a collection of Sass (SCSS) mixins and placeholders for common styling patterns.

You can think of the word Scut as an acronym for SCSS Utitilies.

(Or you can think of the word in other ways: Scut will do your scut work; let Scut be your scut monkey. (Be careful how you use this word in polite company. To learn its many and colorful connotations, have a look at Wiktionary.))

BE AWARE: Scut is in early stages, under active development, unstable — still in SemVer v0. Scut is wide, wide open to contributions, and I will be building on it myself. New features will be added and old features might change. So if a part of Scut that you had been using suddenly doesn't work, please check this documentation to see what changed.

Visit the Github repository

Try out Scut at the Scut Playground on Codepen

What makes Scut unique?

Flexible abstractions of common patterns — nothing more

Scut doesn't require additional setup and processessing, doesn't apply any default sizes, spacing, colors, etc. It strives to be as simple and un-opinionated as possible, offering abstract patterns that you can work into your own design without having to worry about overriding anything or setting-up any site-wide parameters.

Scut's primary goal is to save you from typing the same few lines of styling code over and over again to achieve the same effect in different places.

No vendor prefixes

Some other pre-precessor libraries do little more than vendor prefixing. Scut simply relies on Autoprefixer, then looks for other things to do.

Because of this, if you are using Scut, do not forget to install and run Autoprefixer as part of your compilation process.

(I rely on Grunt to make this happen, with the grunt-autoprefixer plugin. Easy as can be.)

@extend placeholders

All mixins that require no arguments — either because they have no arguments or because all their arguments have default values — are paired with placeholder selectors, which are good things to use, when you can.

scut- prefix

To avoid naming collisions — since this collection may be used alongside other libraries, certianly alongside your own rulesets — all mixins and placeholders are prefixed with scut-.

Installation

  1. Get the files. You have the following options:
    • Use Bower (recommended): 
      bower install scut --save-dev
       This will get you the latest release without superfluous files.
    • Download the latest release from Github.
    • Download the repository from Github.
  2. Then, from your own Sass/SCSS files, import _scut.scss. Like this: 
    @import "path/to/scut";

Make sure you import Scut above any other imports that will rely on Scut.

Also, make sure that you install and use Autoprefixer. As mentioned above, Scut does not bother with vendor prefixes — assuming, instead, that you will use Autoprefixer to handle that messy business.

Usage

Having imported _scut.scss, use the mixins and placeholders described below in the manner described below.

If you have any questions about how to use Sass/SCSS, refer to the thorough Sass documentation.

If the Scut documentation below is inadequate and/or upsetting, please file an issue or contact me.

If you have been using Scut, you get a new version and something breaks, please check the documentation again to see if you utility has changed slightly. Scut is still in the unstable stages of v0, so things might change.

And again, make sure that you install and use Autoprefixer if you want vendor prefixes (which you do). Otherwise your compiled CSS will have none.

Contributing

Please do!

Just visit the repository on Github and contribute according to the usual Github methods.

If you have any questions, please file an issue or contact me.

scut-reset

type separate file that must be imported

scut-reset is an additional file that you should import into your Sass/SCSS only if you want it. You may not want it.

I've found myself implementing the same element-specific resets over and over again, project after project — making headers semantic rather than presentational; removing all default button styling, which never gets used; and so on. So I decided to store those resets in an SCSS file. That's what scut-reset is.

Have a look at the file to see all of what scut-reset does and assess whether you would like to include it in your own project.

If you'd like to use scut-reset, simply import the file _scut-reset.scss after your other resets (e.g. Normalize).

@import "path/to/scut-reset";
You should find it in the root directory, alongside _scut.scss.

positioning: coordinates

use-name scut-coordinates
type mixin only
arguments
  1. $coordinates [ default: n n n n ]

    A list: top, right, bottom, and left coordinates. n means "nothing". a is interchangeable with auto. See below.

A shortcut for assigning coordinates.

You may want to use this mixin on its own — but this mixin's primary purpose is to facilitate those below, which will pair your coordinates with a position value.

Some guidlines for this mixin and its followers:

  • n (the default value) signifies "nothing" (like life) — so any coordinate assigned n is simply ignored, creating no rule in compiled CSS.
  • a can abbreviate auto (both will work).
  • All four values must be passed: so use n as needed.

See the examples below.

ratio-box

use-name scut-ratio-box
type mixin, with a default-values placeholder
arguments
  1. $ratio [ default: 100% ]

    Height as a percentage of width.
  2. $inner [ default: ".scut-inner" ]

    CSS selector for the inner element. Include quotation marks around your selector. See below.

Give an element with unknown or fluid width a fixed aspect ratio.

Implementation involves a few prerequisite elements — here's what you need:

  • A container element, whose width (which may be fluid) will determine the width of your ratio-box.
  • The ratio-box's outer element, to which the mixin applies.
  • The ratio-box's inner element, whose selector must be passed to the mixin as the $inner (second) argument, and whose content will manifest our special styling. The inner element should be an immediate child of the outer element (CSS selector >, the "child combinator").

The examples should demonstrate.

Example

HTML
<p>These containers are different widths, but the ratio-boxes maintain a 16:9 ratio.</p>
<div class="eg-ratio--c m-1">
  <div class="eg-ratio--outer">
    <div class="eg-ratio--inner">
      <!-- Content! -->
    </div>
  </div>
</div>
<div class="eg-ratio--c m-2">
  <div class="eg-ratio--outer">
    <div class="eg-ratio--inner">
      <!-- Content! -->
    </div>
  </div>
</div>
<p>And the same principle applies to these squares:</p>
<div class="eg-ratio--c m-3">
  <div class="eg-ratio--square">
    <div class="scut-inner">
      <!-- Content! -->
    </div>
  </div>
</div>
<div class="eg-ratio--c m-4">
  <div class="eg-ratio--square">
    <div class="scut-inner">
      <!-- Content! -->
    </div>
  </div>
</div>
SCSS
.eg-ratio--outer {
  // 9 / 16 * 100 = 56.25%
  @include scut-ratio-box(56.25%, ".eg-ratio--inner");
}
.eg-ratio--inner {
  background: gray;
}
.eg-ratio--c {
  margin-top: 1em;
  &.m-1,
  &.m-3 {
    width: 200px;
  }
  &.m-2,
  &.m-4 {
    width: 100px;
  }
}
.eg-ratio--square {
  // Default-values placeholder
  @extend %scut-ratio-box;
  .scut-inner {
    background: gray;
  }
}
Result

These containers are different widths, but the ratio-boxes maintain a 16:9 ratio.

v-center: line-height

use-name scut-vcenter-lh
type mixin only
arguments
  1. $height

Vertically center text by equalizing an element's height and line-height.

Although this method is soothingly, reassuringly simple, requiring none of the extra <div>'s that make us all second-guess ourselves, it is also limited. This method only works for inline content (usually text) that will not exceed a single line. So it's good for buttons and headings with backgrounds and things like that.

If you need to vertically center something else, try one of the mixins below.

  @include scut-vcenter-lh(3em);
  background: gray;
  display: inline-block;
  padding: 0 1em;
}

v-center: inline-block

use-name scut-vcenter-ib
type mixin, with a default-values placeholder
arguments
  1. $inner [ default: .scut-inner ]

    CSS selector for the inner element. Include quotation marks around your selector. See below.

Vertically center an inner element with the inline-block method.

This method requires a container and an inner element. The mixin applies to the container; the to-be-centered inner element is passed (by its selector) as an argument.

Any to-be-centered element should be an immediate child of the container (container > inner).

A word of caution: Because inline-block results in some unpredictable word-like spacing, this method doesn't always work well if you require exact widths (or if your inner element has a 100% width). If you require exactitude and nothing stands in the way of giving your container display: table, try that vertically centering via table display.

Example

HTML
<div class="eg-vcenter-ib m-default">
  <p class="scut-inner">
    A vertically centered element, with the default classname "scut-inner".
  </p>
</div>
<div class="eg-vcenter-ib m-variant">
  <p class="eg-vcenter-ib--inner">
    A vertically centered element, with its own special classname.
  </p>
</div>
SCSS
.eg-vcenter-ib {
  border: 5px solid black;
  padding: 0 1em;
  &.m-default {
    // Default-values placeholder
    @extend %scut-vcenter-ib;
    height: 120px;
  }
  &.m-variant {
    @include scut-vcenter-ib(".eg-vcenter-ib--inner");
    height: 200px;
    margin-top: 1em;
  }
}
Result

triangle

use-name scut-triangle
type mixin, with a default-values placeholder
arguments
  1. $direction [ default: right ]

    The direction that the triangle points: up, down, right, left. Or, for right triangles, the corner of the right angle: up-right, up-left, down-right or down-left.
  2. $size [ default: 1.4em 0.75em ]

    One or two values determining the size of the triangle. To make a 45-45-90 triangle, provide one value. Otherwise, provide two, translating to width and height.
  3. $color [ default: inherit ]
  4. $pseudo [ default: false ]

    Make the triangle a pseudo-element by adding the rule content: "".

Create a wide variety of CSS triangles.

(The default placeholder extension points right.)

The mixin can produce equilateral and isosceles triangles pointing up, down, left, and right; or right triangles with their right angles in the upper right, upper left, lower right, and lower left corners.

When choosing width and height values, consider how their proportion will determine the form. For example, to create an equilateral triangle, you'll want your height value to be ~140% of your width value (width * √2). Anybody remember high school geometry?

Example

HTML
<p>
  <span class="eg-triangle m-default"></span>
  <span class="eg-triangle m-equilateral"></span>
  <span class="eg-triangle m-corner-1"></span>
  <span class="eg-triangle m-corner-2"></span>

reverse italics

use-name scut-reverse-italics
type mixin, with a default-values placeholder
arguments
  1. $elements [ default: false ]

    A list (not comma separated) of CSS selectors for elements that you do not want italicized (<em>, <cite>, and <i> are already taken care of: see below). Include quotation marks around your selectors.

Italicize a section of text while de-italicizing those elements that would normally be italicized. Stand the world on its head.

This is a common typographical practice: in non-italicized text, italics provide mild stress; but in italicized text, it is the non-italicized text that is stressed.

The elements <em>, <cite>, and <i>, which browsers typically italicize, are by default de-italicized. To de-italicize more elements, include some selectors in a list that you pass to the mixin.

Example

HTML
<p class="eg-reverse-italics m-default">DEFAULT:
  <em>Some &lt;em&gt; text</em> sit amet, consectetuer adipiscing elit.
  <cite>Some &lt;cite&gt; text</cite>, aenean commodo ligula eget dolor.
  <strong>Some &lt;strong&gt; text</strong> Cum sociis natoque penatibus
  <i>some &lt;i&gt; text</i> et magnis dis parturient montes, nascetur ridiculus mus
  <b>some &lt;b&gt; text</b>.
  <span class="no-italics">And a &lt;span&gt; with the classname "no-italics".</span>
</p>
<p class="eg-reverse-italics m-variant">VARIANT:
  <em>Some &lt;em&gt; text</em> sit amet, consectetuer adipiscing elit.
  <cite>Some &lt;cite&gt; text</cite>, aenean commodo ligula eget dolor.
  <strong>Some &lt;strong&gt; text</strong> Cum sociis natoque penatibus
  <i>some &lt;i&gt; text</i> et magnis dis parturient montes, nascetur ridiculus mus
  <b>some &lt;b&gt; text</b>.
  <span class="no-italics">And a &lt;span&gt; with the classname "no-italics".</span>
</p>

}

use-name is the name of the utility's mixin and placeholder, including the scut- prefix, as they should appear in your Sass/SCSS. This should be illustrated in the examples.

type tells you how you can invoke the utility. Is it only available as a mixin (e.g. HD breakpoint)? Or as a mixin and a placeholder that always do the same thing (e.g. clearfix)? Or as a mixin and a placeholder that passes default values (e.g. indented paragraphs)? Or as a mixin with some extension placeholders for common variations (e.g. punctuated list)?

arguments lists the arguments, in order, that you can pass to the utility's mixin. To learn more about Sass/SCSS mixin arguments have a look at the Sass documentation.

If no default value is listed, that means there is no default value, so you'll have to provide a value every time you use the mixin.
