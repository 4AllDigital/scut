coordinates

type mixin only
use-name scut-coordinates
arguments
  1. $coordinates [ default: n n n n ]

    A list: top, right, bottom, and left coordinates.

A shortcut for assigning coordinates.

You may want to use this mixin on its own — but you probably want to use one of the extensions below, which will pair your coordinates with a position value.

Some guidlines: n (the default value) indicates "nothing" — so the coordinate is ignored, no rule is created. a can abbreviate auto (both will work). All four values must be passed: so use n as needed. See the examples below.

Example

SCSS
.eg-absolute {
  border: 5px solid black;
  height: 150px;
  width: 150px;
  position: relative;
}
.eg-absolute--i {
  background: gray;
  &.m-left {
    @include scut-absolute(0.5em n n 0.5em);
  }
  &.m-right {
    @include scut-absolute(n 0.5em 0.5em n);
  }
}

ratio box

type mixin, with default-values placeholder
use-name scut-ratio-box
arguments
  1. $ratio [ default: 100% ]

    Height as a percentage of width.
  2. $inner [ default: .scut-inner ]

    CSS selector for the inner element. See below.

Give an element with unknown or fluid width a fixed aspect ratio.

Implementation involves a few prerequisite elements — here's what you need:

  • A container element, whose width (which may be fluid) will determine the width of your ratio-box.
  • The ratio-box's outer element, to which the mixin applies.
  • The ratio-box's inner element, whose selector must be passed to the mixin as the $inner (second) argument, and whose content will show the styling.

The examples should demonstrate.

Example

HTML
<p>The containers are different widths, but the ratio-boxes maintain a 16:9 ratio.</p>
<div class="eg-ratio--c m-1">
  <div class="eg-ratio--outer">
    <div class="eg-ratio--inner">
      <!-- Content! -->
    </div>
  </div>
</div>
<div class="eg-ratio--c m-2">
  <div class="eg-ratio--outer">
    <div class="eg-ratio--inner">
      <!-- Content! -->
    </div>
  </div>
</div>
SCSS
.eg-ratio--outer {
  // 9 / 16 * 100 = 56.25%
  @include scut-ratio-box(56.25%, ".eg-ratio--inner");
}
.eg-ratio--inner {
  background: gray;
}
.eg-ratio--c {
  margin-top: 1em;
  &.m-1 {
    width: 200px;
  }
  &.m-2 {
    width: 100px;
  }
}
Result

The containers are different widths, but the ratio-boxes maintain a 16:9 ratio.

vertically center

vertically center via line height

type mixin only
use-name scut-vcenter-lh
arguments
  1. $height

Vertically center text by equalizing an element's height and line-height.

Although this method is nice and simple, requiring no extra <div>s, it is limited: it only works for inline content (i.e. text) that will not exceed a single line. So it's good for buttons and headings with backgrounds and things like that.

If you need to vertically center something else, try one of the mixins below.

Example

SCSS
.eg-vcenter-lh {
  @include scut-vcenter-lh(3em);
  background: gray;
  display: inline-block;
  padding: 0 1em;
}
Result

Vertically centered text.

vertically center via inline block

type mixin, with default-values placeholder
use-name scut-vcenter-ib
arguments
  1. $inner [ default: .scut-inner ]

    CSS selector for the inner element. See below.

Vertically center an inner element with the inline-block method.

This method requires a container and an inner element. The mixin applies to the container; the to-be-centered inner element is passed (by its selector) as an argument.

A word of caution: Because inline-block results in some unpredictable word-like spacing, this method doesn't work well if you require exactitude (e.g. your inner element has a 100% width).* If you require exactitude and nothing stands in the way of your container element having display: table, try that method (below).

Example

HTML
<div class="eg-vcenter-ib">
  <p class="eg-vcenter-ib--inner">
    A vertically centered element.
  </p>
</div>
SCSS
.eg-vcenter-ib {
  @include scut-vcenter-ib(".eg-vcenter-ib--inner");
  border: 5px solid black;
  height: 100px;
}
.eg-vcenter-ib--inner {
  background: gray;
}
Result

A vertically centered element.

References

  • CSS-Tricks: "Centering in the Unknown"

triangle

type mixin, with default-values placeholder
use-name scut-triangle
arguments
  1. $direction [ default: right ]

    The direction that the triangle points: up, down, right, left. Or, for right triangles, the corner of the right angle: up-right, up-left, down-right or down-left.
  2. $size [ default: 1.4em 0.75em ]

    One or two values determining the size of the triangle. To make a 45-45-90 triangle, provide one value. Otherwise, provide two, translating to width and height.
  3. $color [ default: inherit ]
  4. $pseudo [ default: false ]

    Make the triangle a pseudo-element by adding the rule content: "".

A flexible mixin to create a variety of CSS triangles. (And a defult placeholder extension pointing right.)

The mixin can produce equilateral and isosceles triangles pointing up, down, left, and right; or right triangles with their right angles in the upper right, upper left, lower right, and lower left corners.

When choosing width and height values, consider how their proportion will determine the form. For example, to create an equilateral triangle, you'll want your height value to be ~140% of your width value (width * √2). Anybody remember high school geometry?

Example

SCSS
.eg-triangle {
  margin-right: 1em;
  &.m-default {
    @extend %scut-triangle;
  }
  &.m-equilateral {
    @include scut-triangle(up, 1em 1.4em, gray);
  }
  &.m-corner-1 {
    @include scut-triangle(up-right, 1em, blue);
  }
  &.m-corner-2 {
    @include scut-triangle(down-left, 2em 1em, red);
  }
}
.eg-triangle-button {
  border: 1px solid;
  border-radius: 5px;
  display: inline-block;
  padding: 0.1em 0.5em;
  &:after {
    margin-left: 0.3em;
  }
  &.m-down:after {
    @include scut-triangle(down, 0.7em 1em, $pseudo: true);
    margin-bottom: 0.15em;
  }
  &.m-up:after {
    @include scut-triangle(up, 0.7em 1em, $pseudo: true);
    margin-bottom: 0.2em;
  }
  &.m-more {
    color: blue;
    &:hover {
      color: red;
    }
    &:after {
      @include scut-triangle($pseudo: true);
    }
  }
}

reverse italics

type mixin, with default-values placeholder
use-name scut-reverse-italics
arguments
  1. $elements [ default: false ]

    A list (not comma separated) of CSS selectors for elements that you do not want italicized (<em>, <cite>, and <i> are already taken care of: see below).

Italicize a section of text while de-italicizing those elements that would normally be italicized.

This is a common typographical practice: in non-italicized text, italics provide mild stress; but in italicized text, it is the non-italicized text that is stressed.

The elements <em>, <cite>, and <i>, which browsers typically render as italicized, are by default de-italicized. To de-italicize more elements, just include selectors in a list that you pass to the mixin.

Example

HTML
<p class="eg-reverse-italics m-default">DEFAULT:
  <em>Some &lt;em&gt; text</em> sit amet, consectetuer adipiscing elit.
  <cite>Some &lt;cite&gt; text</cite>, aenean commodo ligula eget dolor.
  <strong>Some &lt;strong&gt; text</strong> Cum sociis natoque penatibus
  <i>some &lt;i&gt; text</i> et magnis dis parturient montes, nascetur ridiculus mus
  <b>some &lt;b&gt; text</b>.
  <span class="no-italics">And a &lt;span&gt; with the classname "no-italics".</span>
</p>
<p class="eg-reverse-italics m-variant">VARIANT:
  <em>Some &lt;em&gt; text</em> sit amet, consectetuer adipiscing elit.
  <cite>Some &lt;cite&gt; text</cite>, aenean commodo ligula eget dolor.
  <strong>Some &lt;strong&gt; text</strong> Cum sociis natoque penatibus
  <i>some &lt;i&gt; text</i> et magnis dis parturient montes, nascetur ridiculus mus
  <b>some &lt;b&gt; text</b>.
  <span class="no-italics">And a &lt;span&gt; with the classname "no-italics".</span>
</p>
SCSS
.eg-reverse-italics {
  &.m-default {
    @extend %scut-reverse-italics;
  }
  &.m-variant {
    @include scut-reverse-italics(strong b ".no-italics");
  }
