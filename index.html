hd breakpoint

type mixin only
use-name scut-hd-bp
arguments
  1. $ratio [ default: 1.3 ]

    A minimum resolution.

Simple means to write effective resolution-based @media-queries, using -o-min-device-pixel-ratio, -webkit-min-device-pixel-ratio, and min-resolution.

Example

SCSS
.example-hd-bp {
  color: gray;
  @include scut-hd-bp(2.0) {
    color: blue;
  }
}
Result

If you are using a retina screen, this paragraph will be blue — otherwise, a sad, sad gray.

References

  • The involved Github discussion that resulted in HTML5 Boilerplate's HiDPI breakpoint

  • Matt Thomas's post about Wordpress's HiDPI breakpoint (implemented in WordPress core 3.5)

hide text

type mixin and placeholder
use-name scut-hide-text

Include text in the markup while hiding it from view.

This technique is most commonly used to provide (semantic, accessible, SEO-friendly, etc.) text that will be (visually) replaced with an image.

Example

HTML
<p>The following image is hiding text:</p>
<div class="example-hide-text">This text is hidden</div>
SCSS
.example-hide-text {
  @extend %scut-hide-text;
  background: url(images/bird.jpg) no-repeat;
  background-size: contain;
  height: 100px;
}
Result

selected

type mixin only
use-name scut-selected
arguments
  1. [ default: true ]

    Will the rules apply to the :active state?

Apply rules to :hover, :focus, and (optionally) :active states at the same time.

Don't forget to add a tabindex to any element that will need it for :focus.

Example

SCSS
.example-selected {
  background-color: gray;
  padding: 3px 10px;
  @include scut-selected {
    background-color: blue;
    color: white;
  }
}
Result

The following button turns blue in all three states

fill

type mixin and placeholder
use-name scut-fill

Make an element fill its parent.

Note: The parent-to-be-filled must be positioned — so add position: relative if the parent's position is not already absolute or fixed.

Example

HTML
<div class="example-fill--parent">
  Parent provides the border
  <div class="example-fill--child">
    Child provides the background
  </div>
</div>
SCSS
.example-fill--parent {
  border: 3px solid black;
  color: red;
  height: 100px;
  padding: 1em;
  position: relative;
}

.example-fill--child {
  @extend %scut-fill;
  background: rgba(0,0,0,0.4);
  padding: 2.5em 1em 0;
}
Result
Parent provides the border
Child provides the background

punctuated list

type mixin with extension placeholders
use-name scut-list-punctuated
arguments
  1. $divider [ default: ", " ]

    Choose your punctuation.
  2. $display [ default: inline ]

    Change to inline-block if you don't want your list items spanning lines.
  3. $in-media [ default: false ]

    Go with true if using the mixin within a @media-query, to avoid a futile attempt to @extend within it.

Keep your HTML semantic and your punctuation traditional.

Since the most common punctuated list uses commas, commas are default and the comma list is available as its own placeholder:

  • %scut-list-comma

Example

SCSS
.example-punctuated-list {
  @include scut-list-punctuated;
  // or @extend %scut-list-comma;
}
.example-semicolon-list {
  @include scut-list-punctuated("; ");
}
.example-avant-garde-list {
  @include scut-list-punctuated(" . ");
  text-transform: lowercase;
}

positioning

coordinates

type mixin only
use-name scut-coordinates
arguments
  1. $coordinates [ default: n n n n ]

    A list: top, right, bottom, and left coordinates.

A shortcut for assigning coordinates.

You may want to use this mixin on its own — but you probably want to use one of the extensions below, which will pair your coordinates with a position value.

Some guidlines: n (the default value) indicates "nothing" — so the coordinate is ignored, no rule is created. a can abbreviate auto (both will work). All four values must be passed: so use n as needed. See the examples below.

absolute positioning

type mixin only
use-name scut-absolute
arguments
  1. $coordinates [ default: n n n n ]

    See information about coordinates above.

Absolutely position an element and assign its coordinates, all in one line of SCSS. You'll save years of your life.

Example

SCSS
.example-absolute {
  border: 5px solid black;
  height: 150px;
  width: 150px;
  position: relative;
}
.example-absolute--i {
  background: blue;
  color: white;
  &.m-left {
    @include scut-absolute(0.5em n n 0.5em);
  }
  &.m-right {
    @include scut-absolute(n 0.5em 0.5em n);
  }
}
Result
